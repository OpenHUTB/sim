\keywordsen{Human-Drive Interaction, VS Code Extensions, Self-Driving Car Maneuvering Algorithm Refinement, Fault Identification by Static Analysis, Uniform Computer Programming Workspace.}
\begin{abstracten}

With the rapid development of self-driving cars and intelligent transportation systems, HMI simulators have also been applied to all parts of the automobile industry. Traditional engineering workflows are not efficient, there are many bugs in the fixing process, verification is lengthy, and the configuration is complicated. A high-productivity prototype and assessment infrastructure for driver-vehicle interfaces in Microsoft Visual Studio Code have been created and used in this paper. To provide support for all-round development of the Design of the human-system test bench architecture and multi-carrier protocol standards, intelligent code recommendation, static error analysis, virtual parameter execution switch, dynamic metric surveillance board, parallel testing assets, etc., have been added. The Module is simple to extend and can be used in other places as well. Based on the study of the above experience, the support tool has increased the construction efficiency of vehicle-human contexts by about 40\% and achieved a bug-detection accuracy of more than 95\%; therefore, it has reduced the skill requirements significantly, shortened the officialisation period, and improved the overall performance of the collaborative group.

\end{abstracten}
