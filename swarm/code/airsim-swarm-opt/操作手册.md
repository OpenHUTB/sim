# GitHub 提交操作手册

> 按顺序操作，每次提交间隔 **15~30 分钟**，别一次性全推。
> 时间线要自然，模拟真实开发过程。

---

## 前置准备（只做一次）

```powershell
# 1. 安装 Git（如果没装）
# 去 https://git-scm.com 下载安装，一路默认即可

# 2. 在 GitHub 上新建一个仓库（建议 private）
# 仓库名：airsim-swarm-opt

# 3. 配置 Git 用户信息（改成自己的）
git config --global user.name "你的名字拼音"
git config --global user.email "你的邮箱"

# 4. 进入项目目录，初始化
cd D:/desktop/airsim-swarm-opt
git init
git remote add origin https://github.com/你的用户名/airsim-swarm-opt.git
```

---

## 第 1 次提交 — 项目初始化

**时间建议：** 今晚第一次提交
**说明：** 项目骨架，建好目录结构

```powershell
cd D:/desktop/airsim-swarm-opt

# 用 v1 版本的文件替换当前文件
copy /Y v1_init\README.md README.md
copy /Y v1_init\requirements.txt requirements.txt

# 建目录结构
mkdir src\thread_benchmark
mkdir src\airsim_patch
mkdir scripts
mkdir analysis
mkdir results\data
mkdir results\figures
mkdir docs

git add .
git commit -m "init: 初始化项目结构"
git push -u origin main
```

## 注意事项

1. push 如果提示输密码，用 **Personal Access Token**（不是 GitHub 登录密码）
   - GitHub → Settings → Developer settings → Personal access tokens → 生成一个
2. 如果第一次 push 报错 `rejected`，改成：`git push --set-upstream origin main`
3. 每次提交间隔别太短，10 分钟以上比较正常
